Detailed Description

Structure that represents a USB device internally to the USBH library. The application will reference this structure as a handle.

Field Documentation

◆ address

uint8_t USBH_device::address

The bus address which is assigned to this device. Values are 0-127.

◆ children

USBH_device* USBH_device::children

Pointer to a list of child USBH_device structures. This is used when this device is a hub and there are devices connected to the ports. This is an upwards link to children.

◆ configuration

uint8_t USBH_device::configuration

Configuration number for this device set with SET_CONFIGURATION. A non-zero value means that it is operational.

◆ endpoint0

USBH_endpoint* USBH_device::endpoint0

Pointer to a structure of USB_endpoint for the control endpoints for this device. This is common to all interfaces of this device.

◆ enumValue

uint16_t USBH_device::enumValue

Marker to say that this device has been enumerated. This forms a fairly unique number which increments at each pass of enumeration. A zero value is not enumerated and non-zero value can be used to detect changes in enumeration and invalid handles. Size is 12 bits.

◆ hubPollTimeout

uint16_t USBH_device::hubPollTimeout

HUB: Poll interval timeout. This is approximated in milliseconds, but the bInterval field in the interface descriptor depends on the device speed. It is not used in the library - just calculated and kept for reference.

◆ hubStatusChange

uint8_t USBH_device::hubStatusChange[16]

HUB: Status change bitmap. This is the last value read from the hub's Status Change Endpoint. There can be (in theory) 256 ports on a hub (although only 127 devices).

◆ interfaces

USBH_interface* USBH_device::interfaces

Pointer to a list of USBH_interface structures for interfaces of this device. A NULL pointer indicates the end of the interface list. This is a downward link to children.

◆ next

USBH_device* USBH_device::next

Pointer to the next USBH_device. A NULL pointer indicates the end of the device list. This is a horizontal link to peers.

◆ parent

USBH_device* USBH_device::parent

Pointer to the parent USBH_device. A NULL pointer indicates that this device is connected to the root hub. This is an upwards link to a parent hub.

◆ parentPort

uint8_t USBH_device::parentPort

The port number which this device is connected to on the parent hub. For devices on the root hub this will be zero.

◆ portCount

uint8_t USBH_device::portCount

HUB: The number of downstream ports on this device. This is used when this device is a hub and there are devices connected to the ports.
